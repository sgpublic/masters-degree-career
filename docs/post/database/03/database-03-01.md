# 3.1 关系模型的组成要素及其优缺点

## 3.1.1 数据模型的组成要素

+ 数据结构：数据及数据间的联系，逻辑和物理结构等
+ 数据操作：操作类型、操作方式等
+ 约束条件：数据的语义，数据间的约束等

## 3.1.2 关系模型的优缺点

优点：

+ 用二维表表示数据，**结构简单、概念清晰**。
+ 建立在严格的数学基础之上，**理论基础完善**。
+ 支持**逻辑独立性和物理独立性**，便于开发和维护。
+ **使用非过程化查询**，用户只描述“做什么”，不关心“怎么做”。
+ **使用统一的数据访问接口**，通用性强。

缺点：

+ 查询效率在复杂场景下不高。
+ 对复杂数据类型支持不足。
+ 不适合高并发、海量数据的极端场景。
+ 模式相对固定，灵活性较差。
